# CLAUDE.md

このファイルは Claude Code がこのプロジェクトで作業する際の指針です。

## プロジェクトの目的

縄文土器片の3Dスキャン（PLYメッシュ）から、**縄（撚り紐）を転がして付けた縄文圧痕の繰り返し周期**を検出する。

縄を転がすと撚り目が一定間隔で並ぶため、ある走査線上の深さは周期的になる。
「縄が一周する」= 撚りパターンが1周期ぶん進むこと。この1周期に相当する
**間隔（ピッチ）と方向（縄の進行方向）** を3Dデータから推定するのが最終目標。

### 最終ゴール（最重要）
**PLYファイルを渡すだけで、人手の入力なしに周期を自動特定する**こと。
完成形は次のようなCLIを想定:

```
jomon_period input.ply
→ 周期ベクトル（基準面上の dx, dy）/ ピッチ（mm）/ 縄の進行方向（角度）/ 信頼度 を出力
```

ユーザーによる点クリックや事前の正解座標は**最終成果物には不要**。
2次元自己相関による周期推定はもともと人の入力を必要としないため、
全自動化しても手法の主軸は変わらない。

### 点クリックの位置づけ（開発時のみ）
3Dビューアでの2点クリックは**開発・検証フェーズ限定の答え合わせ手段**。
「自動推定した周期ベクトル」が「人が同じに見えると判断した2点間ベクトル」と
一致するかを確かめるためだけに使う。完成形からは外してよい（オプション扱い）。

## 重要な制約・前提（必ず守ること）

### 入力データ
- 入力は単一の **バイナリPLYメッシュ**（`format binary_little_endian 1.0`）。
- 頂点プロパティは **x, y, z（float32）のみ**。**頂点カラー（RGB）は無い**。
  → 色は一切使えない。**形状（凹凸）だけで判定する**こと。
- 面は三角形リスト（`property list uchar int vertex_indices`）。
- 規模: 頂点 約741.7万、面 約1483.5万。**742万頂点級を前提に処理を設計**すること
  （安易な全頂点二重ループ禁止。空間分割・対象領域切り出し・サンプリングを使う）。
- スケール: バウンディングボックスは約 86.6 × 63.7 × 14.9（単位はmmと推定）。
  撚り目ピッチは数mmオーダーと想定。
- サンプルファイルのパスはユーザー環境依存。**パスは引数で受け取り、ハードコードしない**。

### 力依存問題（設計の核心）
- 人が手で押し付けるため、圧痕の **絶対的な深さは場所によって変わる**。
- したがって **深さの絶対値の大小で判定してはいけない**。
- 判定は「山と谷が繰り返す **間隔（周期）**」と「相対的な凹凸の **形・並び**」で行う。
  振幅（深さ）が変わっても周期はほぼ不変、という性質を利用する。

### 開発環境
- 言語: **C++（C++17 以上）**。
- OS: **Windows と macOS の両方でビルド・動作**すること。Linux も壊さない。
- ビルド: **CMake**。依存解決は **vcpkg**（マニフェストモード推奨）。
- OS依存・コンパイラ依存のコードは避け、書く場合は必ず分岐を明示。

## 推奨ライブラリ構成（変更する場合はユーザーに確認）
- PLY読み込み: **happly**（ヘッダオンリ、軽量、クロスプラットフォーム、FetchContent）
- 線形代数・数値処理: **Eigen**（vcpkg）
- 可視化・点クリック取得: **Polyscope**（FetchContent。Open3Dはvcpkgに存在しないため不採用）
- （代替案: すべてを **PCL** で統一も可能だが重い。明確な要望が無い限り上記を優先）

## 処理パイプライン（実装方針）

1. **読み込み**: happly でメッシュを読む。頂点・面を Eigen 行列へ。
2. **対象面の切り出し**: 破片には縁・側面・破断面が含まれる。法線が揃った主たる
   文様面だけを抽出する（法線クラスタリング、または最大の平坦寄り領域）。
3. **深さマップ化**:
   - 対象面を平面 or なめらかな曲面に近似し、その面を基準に各点の符号付き距離（深さ）を出す。
   - 基準面上に格子を張り、深さを2Dの高さ画像（height map）としてサンプリング。
4. **detrend（重要）**: 土器の湾曲による低周波うねりを除去（局所平均を引く＝ハイパス）。
   撚り目周期はこれより細かいので、低周波を消してから周期解析する。
5. **周期ベクトル推定（主軸）**:
   - 高さ画像に **2次元自己相関** をかけ、原点以外の最初の有意ピークの位置 =
     **周期ベクトル（間隔＋方向）** を得る。
   - これにより縄の進行方向（撚り目の並ぶ向き）と1周期のピッチが同時に出る。
   - 単一ラインは不安定なので2D自己相関を基本とし、必要なら複数平行ラインの
     1次元自己相関/FFTの中央値で補強する。
6. **出力（完成形）**:
   - 周期ベクトル（基準面上の dx, dy）、ピッチ（mm）、縄の進行方向（角度）、
     信頼度（自己相関ピークの鋭さ等）を出力。人手の入力なしでここまで到達する。
7. **検証（開発時のみ・任意）**:
   - 3Dビューアでユーザーが2点をクリック → 3D座標を取得。
   - 2点間ベクトルを基準面上に投影し、自動推定した周期ベクトルの整数倍と一致するか比較。
   - 一致度・誤差を出力。これは手法の妥当性確認用で、完成形には含めなくてよい。
8. **補助手段（力依存が問題になったとき用）**:
   - **局所正規化相互相関（ZNCC）**: 指定2点周辺の小領域が同一パターンか定量評価。
     局所的に平均・コントラストを揃えるので力の強弱にロバスト。
   - **勾配/稜線方向の特徴量**: 深さ絶対値でなく溝の傾き・向きで照合。

## マイルストーン（この順で進める）
1. PLY読み込み + 基本可視化（クロスプラットフォームでビルドが通ること）。
2. 対象面切り出し + 深さマップ化 + detrend。
3. 2次元自己相関による周期ベクトル推定（**ここで全自動の心臓部が完成**）。
4. CLI化: `jomon_period input.ply` で周期・ピッチ・方向・信頼度を出力（**最終ゴール形**）。
5. （検証用・任意）2点クリックツールを別実行ファイルとして用意し、
   自動推定結果と人の判断が一致するか答え合わせ。
6. （必要なら）ZNCC / 勾配特徴による補強で精度・頑健性を上げる。

検証用の点クリック（旧マイルストーン2）は最終成果物に必須ではないため、
本筋（1→2→3→4）を先に通すこと。

## コーディング規約・注意
- 742万頂点を扱うのでメモリと計算量に注意。コピー削減、Eigen の Map 活用、
  対象領域の早期切り出しを徹底。
- パラメータ（格子解像度、detrendの窓サイズ、自己相関のピーク探索範囲など）は
  ハードコードせず、設定または引数で外出しする。
- 単位はmm想定だが、データに依存しないようスケールはバウンディングボックスから
  相対的に決める。
- まず動く最小実装を出し、各マイルストーンごとに可視化で結果を確認できるようにする。
